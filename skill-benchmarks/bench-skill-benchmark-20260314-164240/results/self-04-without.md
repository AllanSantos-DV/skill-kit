# Evaluation Rubric for Git Workflow Management

## Correctness
- Does the output correctly describe Git workflow best practices?
- Are the commands and configurations accurate?
- Is the advice technically sound?

## Completeness
- Does it cover conventional commits?
- Does it mention branch naming?
- Does it include PR review guidance?
- Does it address rebase vs merge?
- Does it discuss merge conflicts?

## Pattern Adherence
- Does it follow established Git conventions?
- Is the advice consistent with industry standards?
- Does it recommend structured approaches?

## Edge Cases
- Does it handle unusual situations?
- Does it address common mistakes?
- Does it consider team collaboration scenarios?
