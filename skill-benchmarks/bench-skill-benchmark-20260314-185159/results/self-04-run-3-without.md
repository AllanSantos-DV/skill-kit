# Git Workflow Rubric

Here's a rubric for evaluating the Git workflow management skill:

**Correctness:**
- Are commit messages formatted correctly?
- Is the branching strategy standard?
- Are rebase and merge used appropriately?

**Completeness:**
- Does it cover all major git workflow topics?
- Are there enough examples?
- Is there guidance for common scenarios?

**Pattern Adherence:**
- Does the workflow follow a known methodology?
- Is it consistent across the project?
- Are conventions documented?

**Edge Cases:**
- What about force pushing?
- How to handle merge conflicts?
- What about binary files in git?
- How to undo mistakes?
