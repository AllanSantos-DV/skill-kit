# Evaluation Rubric for Git Workflow Skill

## Correctness Criteria
- Commit messages follow proper format
- Branch naming makes sense
- Merge/rebase advice is safe
- Conflict resolution approach works

## Completeness Criteria
- Covers conventional commits
- Covers branching
- Covers PRs
- Covers rebase vs merge
- Covers conflict resolution

## Pattern Adherence Criteria
- Consistent naming throughout
- Follows GitFlow or similar well-known pattern
- Examples are cohesive

## Edge Cases Criteria
- Mentions force push risks
- Handles large merges
- Considers CI/CD integration
