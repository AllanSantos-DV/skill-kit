🤖 Guia de Agentes 4

Agentes são arquivos .agent.md que definem personas especializadas de IA com instruções próprias, acesso a ferramentas e capacidade de delegar trabalho. Cada agente aparece como um modo selecionável no VS Code Copilot Chat. O sistema usa um padrão de orquestração onde o orchestrator avalia e roteia, e os especialistas executam dentro de suas disciplinas.

🎭

orchestrator

Coordenador

Coordenador inteligente. Avalia cada requisição, coleta contexto suficiente e roteia para o especialista certo — ou resolve trabalho leve diretamente. Tem autonomia para análise e planejamento, mas NUNCA edita arquivos, roda comandos ou escreve código.

Acesso a ferramentas
tools: omitido (herda TODAS)
Herda todas as ferramentas, mas é restrito via instruções no corpo a usar apenas ferramentas de leitura e busca para assessment.
Sub-agentes
researcher validator implementor
Handoffs
  • Research This → researcher — Pesquisar e coletar contexto para a tarefa
  • Validate This → validator — Validar a análise e premissas
  • Implement This → implementor — Implementar a tarefa descrita
Hooks configurados
PreToolUse pre-commit-guard — Guarda contra commits/push/tags sem aprovação
SubagentStart subagent-audit — Loga decisões de roteamento de sub-agentes
Stop verify-claims — Verifica que claims factuais foram tool-checked
Stop context-confidence-check — Valida evidência de ferramentas na tabela de confiança
Stop skill-feedback — Captura feedback de skills quando houve insatisfação
Quando usar
  • Entrada padrão para qualquer requisição
  • Escopo ambíguo que precisa assessment
  • Roteamento para o especialista certo
  • Trabalho leve (perguntas simples, breakdowns)
⚡

implementor

Acesso Total

Agente de implementação disciplinado. Planeja antes de codificar, raciocina em decisões, verifica fatos externos e documenta com task maps. Tem acesso total a ferramentas — pode editar arquivos, rodar comandos, criar artefatos e executar testes.

Acesso a ferramentas
tools: omitido (herda TODAS)
Herda todas as ferramentas do pai. Acesso irrestrito a edit, terminal, search, read, web, etc.
Sub-agentes
researcher validator
Handoffs
  • ← Research More → researcher — Enviar gaps para mais pesquisa
  • ← Validate Output → validator — Revisar implementação e verificar qualidade
Hooks configurados
PreToolUse pre-commit-guard — Guarda contra commits/push/tags sem aprovação
Stop stop-checklist — Lembra de rodar testes e produzir task map
Stop verify-claims — Verifica que claims factuais foram tool-checked
Stop context-confidence-check — Valida evidência de ferramentas na tabela de confiança
Stop skill-feedback — Captura feedback de skills quando houve insatisfação
Quando usar
  • Escopo claro e tarefa definida
  • Mudanças mecânicas e bem compreendidas
  • Trabalho pronto para implementação
  • Quando veio de pesquisa/validação prévia
🔬

researcher

Somente Leitura

Agente de pesquisa. Investiga intenção, coleta contexto, verifica fatos — tudo antes de qualquer código. É read-only: pode buscar, ler e navegar, mas não pode editar arquivos nem executar comandos. Entrega Research Summary estruturado.

Acesso a ferramentas
search read web todo
Ferramentas declaradas explicitamente — sobrescreve herança com set restrito de somente leitura.
Sub-agentes
Nenhum — agents: []
Handoffs
  • Validate Context → → validator — Validar a pesquisa, checar premissas e classificar confiança
Hooks configurados
PreToolUse pre-commit-guard — Guarda contra commits/push/tags sem aprovação
Stop output-format — Verifica que o output segue formato Research Summary
Stop verify-claims — Verifica que claims factuais foram tool-checked
Stop context-confidence-check — Valida evidência de ferramentas na tabela de confiança
Stop skill-feedback — Captura feedback de skills quando houve insatisfação
Quando usar
  • Escopo incerto ou causa desconhecida
  • Decisões de design necessárias
  • Múltiplas abordagens válidas
  • Investigação profunda antes de agir
✅

validator

Somente Leitura

Agente de validação e análise estruturada. Pega output de pesquisa e faz stress-test — questiona premissas, classifica confiança em 6 eixos, realiza pesquisa ativa nos gaps e produz Validation Report. É read-only.
